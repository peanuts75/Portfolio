Testing tools


Unit 1 - Outcome 3.4

Describe how you would go about testing digital solutions.

For this project, the solution that I’m testing is a website, which is perhaps the easiest solution that can be tested. Since the file we’re testing is a HTML file and most Operating Systems have HTML files assigned to a web browser by default, testing the website is as simple as saving and then running the HTML file.

Just to make the most of the process, I’m also using Google Chrome as the web browser which has a developer console built-in. This allows me to examine the HTML and CSS elements of the webpage at the press of a button (F12). If I want, I can even simulate disabling a fragment of code so that I can more easily see the effect that it has on the webpage.

There’s also a developer’s console which allows you to manipulate elements using javascript, though only for the active session as none of the changes will affect the files of the website.
